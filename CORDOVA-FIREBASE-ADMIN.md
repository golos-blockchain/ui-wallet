# Настройка уведомлений Firebase (FCM)

Чтобы в приложении Android работали уведомления, нужно подключить приложение к Firebase (FCM), а также связать Golos Notify Service с приложением в Firebase.  
При этом от Firebase нужно получить два файла с ключами - google-services.json (зашивается в приложение) и service account (вставляется в конфиг Golos Notify).  
Ниже - полная инструкция, как все это сделать.

1. Заходим в консоль Firebase:
https://console.firebase.google.com/

2. Если не авторизованы в аккаунте Google, то вводим свою почту и пароль.

3. Если после ввода пароля потребуется двухфакторная аутентификация, то привяжите к аккаунту свой номер телефона, и/или установите любое приложение для 2FA, или браузерное расширение для 2FA. Например для браузера Firefox можно установить это расширение: https://addons.mozilla.org/en-US/firefox/addon/auth-helper/ и добавить туда аккаунт Google согласно инструкциям (про "authenticator app"). Затем при входе нужно (после пароля) ввести 6-значный цифровой код, который показывается в этом расширении при открытии иконки на панели справа-вверху.

4. После того как удалось авторизоваться в консоли Firebase, нажмите там кнопку "Create a new Firebase project"

5. Project name: gls-wallet

6. Enable Gemini: можно оставить вкл

7. Google Analytics: можно оставить вкл, на конфиденциальность пользователей это в данном случае не влияет

8. Google Analytics Account: Default

9. Этап "Создание проекта" занимает около 2 минут.

10. После того как проект создастся и вы зайдете в него, нажмите там кнопку "+ Add app" и выберите "Android".

11. Android package name: gls.wallet (должно совпадать с widget id из файла config.xml)

12. Nickname: gls-wallet

13. Внимание: скачайте google-services.json, как только вам это предложат! Этот файл позже нужно будет добавить в приложение при сборке. При этом не пользуйтесь инструкцией от Firebase (она для нативных Android-приложений), а делайте следующее:

13.1. Поместите файл google-services.json в папку ui-wallet/native_core, а если там уже есть такой файл, то замените

13.2. Находясь в каталоге ui-wallet, выполните команду:
```sh
yarn add file:native_core
```

13.3. Файл добавлен. Нажмите в консоли Firebase кнопку Next, чтобы перейти к следующему этапу.

14. Откройте файл ui-wallet/native_core/firebase-bom.gradle и проверьте версию пакета firebase-bom. Если Firebase предлагает более новую версию, то можете обновить, после чего снова выполнить команду из 13.2.

15. Нажмите Continue to console.

16. Подключите приложение к Golos Notify Service. Для этого:

16.1. В консоли Firebase зайдите в настройки приложения (значок шестеренка)

16.2. Зайдите на вкладку Service accounts

16.3. Нажмите Generate new private key. При этом скачается файл с ключами

16.4. В Golos Notify откройте dataserver/config/default.json. В нем должно быть поле firebase_clients, которое представляет собой объект, и в этом объекте должно быть поле wallet_android, значением которого должно стать содержимое этого файла. Пример правильной структуры:
```js
{
    "firebase_clients": {
        "wallet_android": {
            "type": "service_account",
            "project_id": "gls-wallet",
            "private_key_id": "...",
            "private_key": "...",
            // и также все остальные поля из файла
        }
    }
}
```
После этого файл сохраните, сервис перезапустите.

17. Приложение можно собрать согласно обычной инструкции (если оно уже собиралось, то просто выполните yarn run build:mobile). В случае каких-то проблем с Firebase оно будет показывать соответствующие ошибки. Для работы уведомлений надо дать разрешение, оно запрашивается при первом запуске или выдается в настройках.
